Chapter 01

设计根基 — 三层融合

WorkBuddy 设计规范建立在三条轴线的交叉点上：纸本美学（编辑感、温润色调）、 Apple HIG（清晰·顺从·深度）、Material Design 3（自适应、动态色彩）。 最终产物是对美学有坚持、对平台有尊重的统一设计语言。

Axis 1 — 纸本美学
温润 · 克制 · 编辑感

暖色纸色谱、衬线标题、装饰分隔线、手绘线稿图标。内容如印刷物般被精心编排，而非堆砌在发光白墙上。

Axis 2 — Apple HIG
清晰 · 顺从 · 深度

清晰：每种字号都清晰可读；顺从：UI 从不与内容竞争；深度：视觉层级与真实动效传递秩序。

Axis 3 — Material Design 3
自适应 · 动态 · 可访问

Dynamic Color 算法、组件变体系统、48dp 触控标准。确保 Android 端体验一致性。

页面容器与多栏布局

容器不是越窄越优雅——合理利用横向空间，用多栏排版在单屏内呈现更多有效内容。

容器宽度原则
场景最大宽度说明
长文阅读 / 文档680px最佳行长 45–75 字符，保证阅读舒适度
列表/卡片/资讯流1120px双栏网格，充分利用横向空间
仪表盘/数据看板全宽适应弹性网格，不设最大宽度
表单/对话框520px窄表单更专注，减少视觉干扰
双栏网格 · 推荐写法
/* 双栏卡片布局 — 信息流/日报/列表首选 */ .news-grid { display: grid; grid-template-columns: repeat(2, 1fr); gap: var(--sp-md); } /* 移动端自动回退单列 */ @media (max-width: 768px) { .news-grid { grid-template-columns: 1fr; } }
左右留白 · 纸边原则

容器左右各保留 20px（--page-padding-h）留白——既保留纸本美学的"白边呼吸感"，又不挤占内容展示空间。不是空白越多越好，恰到好处的纸边才像一本编辑精良的刊物。

参数旧值新规范理由
页面最大宽度860px1120px+30%，信息密度提升
水平内边距24px20px精简纸边，不喧宾夺主
内容区有效宽度 (1440p)~812px~1080px单屏展示信息量翻倍
列表排版单列纵向双栏报纸式减少滚动，增强浏览效率

Chapter 07

平台适配指南

以 iOS 设计哲学为主，参考 Apple 的「清晰·顺从·深度」三原则。

iOS HIG 核心原则

原则 1 — 清晰 CLARITY
文字始终清晰可读
  • 14pt 以上正文，16pt 为标准
  • 对比度不低于 4.5:1
  • 图标精确、装饰克制
原则 2 — 顺从 DEFERENCE
UI 辅助内容，而非竞争
  • 元素多用半透明和轻量阴影
  • 边框和分隔线尽量细（≤1px）
  • 大面积留白
原则 3 — 深度 DEPTH
视觉层级传递秩序
  • z-index 控制在 3 层内
  • 弹窗/浮层用模糊半透明背景
  • 动效方向暗示导航层级

iOS 导航模式映射

iOS 模式场景Web 实现
iOS Navigation Bar页面标题 + 后退 + 操作按钮固定顶部栏，sticky
iOS Tab Bar3–5 个主要功能入口固定底部栏，最多 5 个图标 + 文字
iOS Modal Sheet表单、详情、临时操作从底部滑入的抽屉面板
iOS Alert / Action Sheet确认 / 选择操作居中弹窗 / 底部选项列表

WebView 中的 iOS 适配要点

  • 禁用用户缩放但保持系统缩放能力：user-scalable=no

Chapter 08

响应式设计

基于 iOS Size Classes 思路，5 个断点覆盖从 iPhone SE 到 Studio Display。所有产出物必须同时适配 PC 端和移动端——用户可能在手机上打开任何一份报告、日报、看板。

断点体系

断点宽度设备参考策略
sm≥ 640pxiPhone (compact)单列、侧边栏隐藏
md≥ 768pxiPad mini (regular)侧边栏显示、双列
lg≥ 1024pxiPad Pro三列、完整导航
xl≥ 1280px桌面显示器多列、内容居中
2xl≥ 1536px宽屏显示器1120px 最大内容宽度、双栏为主

移动端适配策略

双栏 → 单列堆叠

双栏网格在 <768px 时自动降为单列，报纸排版回归移动端可读模式。

侧边栏 → 汉堡菜单

小屏隐藏固定侧边栏，改为滑出式抽屉。

表格 → 横向滚动

数据表格溢出时启用横向滚动。

卡片 → 全宽堆叠

卡片网格从多列降为单列堆叠。

移动端必配清单 · Mobile Readiness Checklist

每份 HTML 产出在交付前，以下 8 项必须全绿。这不是"建议"，是硬性要求。

#检查项要求验证方法
1 Viewport Meta <meta name="viewport" content="width=device-width,initial-scale=1"> 查看 <head> 首行
2 断点覆盖 至少覆盖 ≤768px 和 ≤480px 两个移动端断点 Chrome DevTools 设备模拟
3 字号不低于 14px 移动端正文最小 14px，标题最小 16px；输入框 16px 防止 iOS 自动缩放 Computed Styles 面板
4 触控区域 ≥ 44×44px 所有可点击元素（按钮、链接、图标）触控区域不低于 44px Chrome DevTools 测量
5 无水平溢出 页面宽度不超过 100vw；表格/代码块允许横向滚动但不撑破页面 左右滑动检查
6 图片/媒体自适应 max-width: 100%; height: auto；图表/截图不超出视口 窄屏查看
7 布局降级 多栏→单列堆叠；侧边栏→顶部折叠/隐藏；间距从 32px 缩至 16px 375px 宽度验证
8 Safe Area 安全区 底部内容避开 Home Indicator 区域（env(safe-area-inset-bottom)） iPhone 真机或 Xcode 模拟器

Chapter 10

图标体系 — 编辑线稿风格

所有图标采用统一的手绘线稿风格：24×24 viewBox、1.5–2px 线宽、currentColor 继承文本色、stroke-linecap="round" 圆润端点。 设计灵感来自旧式印刷品的木刻版画插图，与纸本美学主题一脉相承。

图标使用规范

属性值说明
viewBox0 0 24 24统一画布尺寸
线宽stroke-width="1.8"正文使用 1.8，小尺寸可降至 1.5
端点stroke-linecap="round"圆角端点，柔和
连接stroke-linejoin="round"圆角连接
颜色currentColor继承父元素 color
填充fill="none"默认空心，特殊场景可用 fill-opacity
